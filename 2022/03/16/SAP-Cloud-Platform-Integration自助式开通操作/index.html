自助式开通

SAP Business Technology Platform从以前Neo平台时代专项Cloud Foundry架构，带来变化的除了架构上的变化也包括用户对于BTP Service的使用上习惯的更改。

本编主要介绍如何在Cloud Foundry架构下的自助开通SAP的Cloud Platform Integration Suite服务，让客户自己完成服务开通功能，以顺利进入CPIS开发。

操作指南

具体操作会涉及到两大块内容资源分配和权限分配。资源分配很好理解，将全局账户下面的资源分配给具体要使用的子账户，以便能够启用服务时有资源可用。权限分配会涉及到将CPIS服务的权限分配给具体开发、运营的人员。

资源分配和创建实例

前置条件，已经收到有SAP发送的激活邮件，并且已经完成激活操作。错过了激活时间，请使用邮箱重置密码

  1. 进入全局账号查看资源情况
    进入全局账号下面，在授权服务分配的栏位下面查看全局资源情况，确认Integration Suite和Process Integration Runtime是否正确分配。

  2. 创建子账户并且将资源分配给新创建的子账户
    点击账户浏览器，点击创建，在弹窗下面填写必要信息，最后一步点击创建，完成子账户的创建。

  3. 进入子账户，授权CPIS的资源
    进入第二步创建的子账户中，点击授权，配置服务计划。在弹窗中找到Integration Suite和Process Integration Runtime,并且勾选上所有的选项。最后点击添加服务计划，保存配置计划服务。

  4. 创建CPIS实例
    点击左侧栏位服务，进入实例与租用，点击创建，选择Integration Suite服务，点击创建。完成第一步实例创建。

    实例创建之后，最终会在租用栏位下面看到Integration Suite出现已启用的字样。

进入实例完成自开通

  1. 分配Provisioner权限
    选择左栏位安全，点击用户，进入用户主页面。选择当前登陆账号的用户，选择右边页面总出现的分配角色集合。

    在跳出的角色集合中，选择Integration_proivioner，点击分配角色集合。当前用户就有权限进入子开通的权限，能否访问CPIS Provisioning的界面。

  2. 进入Provisioning的应用服务
    点击左侧实例与租用，选择Integration Suite,点击转到应用程序。

    进入Provisoining页面之后，点击Add Capability。

    勾选上需要完成开通的服务如图所示，包括Cloud Platform Integration, API Management。然后一路点击Next，其中有个环境根据自己需求选择Production或者Non-production。最后按下Active按钮。等待后台系统进度条直至完成。

  3. 系统完成开通
    当系统完成开通之后，就可以看到整个关于CPIS的所有所需要的信息，包括登陆CPI所需要的地址，先记下用于后面的登陆使用。在登陆之前，需要进行CPI的应用授权。

  4. CPI应用授权
    选择左栏位安全，点击用户，进入用户主页面。选择当前登陆账号的用户，选择右边页面总出现的分配角色集合。

    勾选上PI的所有角色，分配给当前用户，完成CPI的授权部分。

  5. 进入CPI云端系统
    在第3步系统开通是记下来的CPI URL，放在浏览器中，一会就会看到系统能够正常登陆。最后完成了所需要的系统子开通的全部步奏。
